# アプリ概要・MVP 定義

## 目的

PHP × Laravel を使って、
**認証・認可を含む基本的な CRUD 処理**を実装し、
就活用ポートフォリオとして公開する。

Docker を利用し、
**第三者が開発環境を再現できる構成**にすることも目的とする。

---

## 実装予定の機能（MVP / v1）

### 認証

- ユーザー登録（アカウント作成）
- ログイン / ログアウト
- 登録完了後は自動的にログイン状態にする

---

### 投稿（CRUD）

- 投稿作成：ログインユーザーのみ可能
- 投稿一覧：未ログインでも閲覧可能
- 投稿詳細：未ログインでも閲覧可能
- 投稿編集 / 削除：投稿者本人のみ可能

---

### 認可（Authorization）

- 自分の投稿のみ編集 / 削除可能
- 他人の投稿に対する編集 / 削除は不可
- 他人の投稿では編集 / 削除ボタンを表示しない
- 投稿の作成/編集/削除は auth ミドルウェア配下（未ログインはログイン画面へ）
- ユーザー名 または メールアドレスでログイン可能
- ヘッダーにログイン中のユーザー名を表示
- ※ Breeze 同梱の「パスワードリセット／メール認証／パスワード変更」は MVP では未使用のためルートを無効化しています（必要になったら有効化予定）

---

### バリデーション

- 投稿作成 / 編集時の必須チェック
  - title：必須（255 文字以内）
  - body：必須（10000 文字以内）
- 不正入力時はエラーメッセージを表示

---

### テスト（最低限）

- 未ログイン時の制御（作成 / 編集へのアクセス制限）
- 投稿作成の正常系
- バリデーションエラー
- 認可（他人の投稿を編集 / 削除できない）

## ルーティング構成（MVP）

※ Laravel の `Route::resource('posts')` を使用

- GET /posts
  投稿一覧（公開）

- GET /posts/{post}
  投稿詳細（公開）

- GET /posts/create
  投稿作成画面（要ログイン）

- POST /posts
  投稿保存（要ログイン）

- GET /posts/{post}/edit
  投稿編集画面（投稿者本人のみ）

- PUT / PATCH /posts/{post}
  投稿更新（投稿者本人のみ）

- DELETE /posts/{post}
  投稿削除（投稿者本人のみ）
