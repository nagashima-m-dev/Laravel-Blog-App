# Laravel Blog App（ブログ投稿サイト）

## 目的

PHP × Laravel を使って、
ログイン機能を含む基本的な CRUD 処理を実装し、
就活用のポートフォリオとして公開する。

Docker を利用し、
開発環境を再現できる構成にすることも目的とする。

---

## 実装予定の機能

### 認証（ログイン / アカウント作成）

- ユーザー名 + パスワードによるログイン
- ログイン画面に「アカウント作成はこちら」へのリンクを設置
- アカウント作成（登録）画面はログイン画面とは別で用意
- 登録完了後は、そのままログイン状態にする想定

---

### 投稿（CRUD）

- 投稿作成：ログインユーザーのみ可能
- 投稿一覧：ログインしていないユーザーも閲覧可能
- 投稿詳細：全ユーザー閲覧可能
- 投稿編集 / 削除：投稿者本人のみ可能

---

### 投稿一覧画面

- タイトル・本文を対象としたフリーワード検索
- ソート機能
  - ビュー数の多い順
  - いいね数の多い順
- 記事クリックで詳細画面へ遷移

---

### 投稿詳細画面

- 投稿本文の表示
- ビュー数
  - 詳細画面が表示された回数をカウント
- いいね機能
  - ログインユーザーのみ操作可能
  - 1 ユーザーにつき 1 記事 1 回まで
  - 再クリックで取り消し可能（トグル式）

---

### ユーザーページ（プロフィール）

- ユーザー名の表示
- 該当ユーザーが投稿した記事の一覧表示

---

## 余裕があれば実装したい機能

### お気に入り

- 記事をお気に入り登録できる機能
- ユーザーページからお気に入り記事の一覧を確認できるようにする

## データモデル（案）

実装を進めながら調整する前提の構成案。

### users

- id
- username
- password
- name（表示名）
- created_at
- updated_at

### posts

- id
- user_id
- title
- body
- views
- created_at
- updated_at

### likes

- id
- user_id
- post_id
- created_at
  ※ user_id + post_id にユニーク制約を設定予定

### favorites（余裕があれば）

- id
- user_id
- post_id
  ※ user_id + post_id にユニーク制約

## ルーティング構成（案）

※ 実装段階で命名・構成は調整する可能性あり。

- GET /
  投稿一覧

- GET /posts/{id}
  投稿詳細（表示時に views を加算）

- GET /posts/create
  投稿作成画面（要ログイン）

- POST /posts
  投稿保存（要ログイン）

- GET /posts/{id}/edit
  投稿編集画面（投稿者本人のみ）

- PUT /posts/{id}
  投稿更新（投稿者本人のみ）

- DELETE /posts/{id}
  投稿削除（投稿者本人のみ）

- POST /posts/{id}/like-toggle
  いいねの追加 / 解除（トグル、要ログイン）

- GET /users/{id}
  ユーザーページ
