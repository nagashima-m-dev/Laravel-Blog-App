# ルーティング設計

本アプリでは、Laravel の `Route::resource` を利用して投稿機能のルーティングを構成しています。

---

## 投稿関連ルーティング（MVP）

| メソッド  | URL                | コントローラ | 認証 | 認可     |
| --------- | ------------------ | ------------ | ---- | -------- |
| GET       | /posts             | index        | 不要 | -        |
| GET       | /posts/{post}      | show         | 不要 | -        |
| GET       | /posts/create      | create       | 必要 | -        |
| POST      | /posts             | store        | 必要 | -        |
| GET       | /posts/{post}/edit | edit         | 必要 | 本人のみ |
| PUT/PATCH | /posts/{post}      | update       | 必要 | 本人のみ |
| DELETE    | /posts/{post}      | destroy      | 必要 | 本人のみ |

---

## 認証ルーティング

- ログイン：`/login`
- アカウント作成：`/register`
- ログアウト：`/logout`

※ パスワードリセット・メール認証関連のルートは、MVP では未使用のため無効化しています。

---

## 認可の考え方

- 投稿の作成・更新は FormRequest の authorize() により制御
- 投稿者本人のみ編集・削除可能
- Blade 側でも条件分岐を行い、操作ボタンの表示を制御

---

## ミドルウェア構成

- 投稿の作成・編集・削除は `auth` ミドルウェア配下
- 一覧・詳細は未ログインでもアクセス可能
