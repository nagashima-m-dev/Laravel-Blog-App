# 開発ルール

## ブランチ運用方針

本リポジトリでは、個人開発・ポートフォリオ用途として、シンプルなブランチ運用を採用しています。

### 基本方針

- `main` ブランチは 常に動作する状態を保つ
- 作業は必ず `feature/*` ブランチで行う
- 機能単位でブランチを切り、PR 経由で `main` にマージする

### ブランチ命名規則

- 機能追加・改修

  - `feature/<内容>`
  - 例：
    - `feature/posts-crud`
    - `feature/authentication`
    - `feature/ui-improvement`

- バージョンを意識した機能追加（v1.1 以降）
  - `feature/v1.1-<内容>`
  - 例：
    - `feature/v1.1-like`
    - `feature/v1.1-search`

### MVP 管理について

- MVP 完成時点の状態は、`main` ブランチにマージした上で Git タグ（例：v1.0.0）を付与して管理する
- 必要に応じて `release/v1.0-mvp` ブランチを作成し、保守・参照用として残すことも想定している

## コミットルール

- 1 コミット = 1 作業単位
- コミットメッセージは日本語で記述する
- 例：
  - `投稿のCRUDを実装`
  - `認可（本人のみ編集・削除）を追加`
